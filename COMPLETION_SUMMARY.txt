HOW TO USE IT
================================================================================

START IN THIS ORDER:

1. Start MongoDB
   docker run -d -p 27017:27017 --name mongodb mongo:latest
   OR: mongosh (if running locally)

2. Start Backend Server
   cd backend
   npm install
   npm run dev
   
   You should see:
   [Database] MongoDB connected successfully
   [Backend] Server running on port 3001

3. Seed Database (in new terminal)
   cd backend
   npm run seed
   
   You should see:
   [Seed] Successfully inserted 6 items
   [Seed] Database seeding completed successfully!

4. Start Frontend (in another terminal)
   npm run dev
   
   Available at: http://localhost:5173

5. Visit Admin Dashboard
   Open: http://localhost:5173/admin
   
   You should see:
   - Total Items: 6
   - Pending Review: 3
   - Verified Items: 2
   - Recovered: 1
   - Pending Verification items list
   - Recent Activity feed
   - Verify/Reject buttons (working!)

================================================================================
TROUBLESHOOTING QUICK TIPS
================================================================================

Problem: "Failed to load dashboard data"
Solution: 
  1. Check MongoDB running: mongosh
  2. Check Backend running: curl http://localhost:3001/health
  3. Restart everything

Problem: No items showing (all 0s)
Solution:
  1. Run seed: npm run seed
  2. Refresh dashboard (Ctrl+F5)
  3. Check MongoDB: mongosh → db.items.count()

Problem: Verify button doesn't work
Solution:
  1. Open DevTools (F12)
  2. Check Network tab for /api/items/:id request
  3. Check Console for error messages
  4. Restart backend server

For more help, see TROUBLESHOOTING.md

IMPORTANT NOTES
================================================================================

1. MongoDB Configuration
   - Default: mongodb://localhost:27017/campus-lost-found
   - If using Atlas, update MONGODB_URI in backend/.env.local
   - Special characters must be URL-encoded

2. Database Seeding
   - Run "npm run seed" to add 6 test items
   - Seed clears old data first
   - Safe to run multiple times

3. Backend Restart
   - Frontend changes auto-reload
   - Backend changes require restart (Ctrl+C, npm run dev)

4. Port Requirements
   - MongoDB: 27017
   - Backend: 3001
   - Frontend: 5173
   - All must be available

5. Security Notes
   - Change JWT_SECRET in production
   - Never commit .env.local with real secrets
   - Use MongoDB Atlas IP whitelist
   - Enable authentication on MongoDB
